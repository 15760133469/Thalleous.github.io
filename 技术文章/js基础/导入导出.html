    导入与导出

    Thalleous 6/4/2023 js基础

    说明

    这是js基础的导入导出模块的摘要内容，vuepress提供的语法块

    import和require区别的讲解视频 (opens new window)

    # require/import/export模块

    require和import的区别

    require是同步加载，全部引入，因此性能较低

    import是异步加载，且能按需引入，因此性能更高

    es6的模块化语法

    导入导出语法：export default/export import

    不论是否声明strict mode，导入的模块都会运行在严格模式下

    es6主要导出模块：export default默认导出，export命名导出

    import的几种写法

    1.defaultExport： 将引用模块默认导出的名称。

    ​ 只能导出export default；defaultExport名字可以随意设置。

    2.module-name：要导入的模块。这通常是包含模块的 .js 文件的相对或绝对路径名，可以不包括 .js 扩展名。某些打包 工具可以允许或要求使用该扩展；检查你的运行环境，只允许单引号和双引号的字符串。

    ​ 可以导出export和export default；需要打包成一个对象赋值给重命名变量

    3.name：引用时将用作一种命名空间的模块对象的名称。

    ​ 只能导出default；可以按需导出

    4.alias，aliasN： 将引用指定的导入的名称。

    ​ 重命名导出（前者为原名字，后者为新名字）

    import { export as alias }

    5.作为副作用导入

    import 'module name'

    import('./myModule.js').then(res => {
	// res为导入的内容
})

    import { export1, export2 } from 'module-name'

    import和webpack的关系

    静态（同步）引入的import经过webpack打包会变成Map对象，key为路径，value为可执行函数；

    异步引入的import会被单独打成一个包，采用动态加载的方式。

    拓展：webpack+import()可以实现Vue组件的懒加载（路由懒加载同理）

    <template>
    <div>
       <Child1></Child1>
       <Child2></Child2>
    </div>
</template>
<script>
export default{
components:{
   Child1:()=>import(./child1.vue),
   Child2:()=>import(./child2.vue)
 }
} 
</script>

    export和export default的使用场景

    export一般是混合多种类导出

    export default导出的只有一个对象、函数声明、变量（函数表达式、变量声明的）字面量

    Vue单组件SFC为什么会使用export default {}

    单文件组件(.vue文件)将被vue-loader解析组装成一个ES Module，默认导出一个Vue.js 组件选项的对象；

    Vue2使用的是option Api（选项API） ，export default{}导出的是对象，这个对象是 Vue 实例的选项对象，以便于在其它地方可以使用 import 引入。
