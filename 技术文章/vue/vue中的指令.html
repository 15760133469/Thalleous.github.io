    # vue中的指令

    v-model的作用和原理

    作用：

    ​ 用于表单元素实现双向数据绑定

    原理：

    ​ v-bind绑定value属性的值

    ​ v-on绑定input事件监听到函数中，函数会获取最新的值，赋值给绑定的属性中

    v-bind的作用

    元素的属性动态绑定值，可以缩写为:

    v-model和v-bind的区别

    v-model是双向绑定，v-bind是单向绑定，其中v-bind是Model -> View，数据调度页面

    v-for动态渲染

    v-for 指令基于一个数组来渲染一个列表。建议设置key值，并且保证每个key值是独一无二的，这便于diff算法（页面渲染更新）进行优化

    v-show显示与隐藏

    v-show的原理是动态为元素添加或移除 display 样式，来实现元素的显示(display:block)和隐藏(display:none)

    v-if显示与隐藏

    v-if 的原理是动态的在DOM树上创建或移除元素，实现元素的显示和隐藏

    v-else-if和v-else需要和v-if连用，否则模板编译会报错

    v-show和v-if的区别

    1.v-if操作dom节点；v-show操作css的display

    2.v-if有一个局部的编译/卸载过程；v-show仅仅是css切换

    3.v-if是惰性的，如果初始为false则完全不编译，只有首次为真时才会局部编译；v-show则不论初始值为true还是false都会编译

    4.v-if有更高的切换消耗，v-show有更高的初始渲染消耗

    v-for和v-if的优先级

    • 在vue2中，v-for的优先级高于v-if

      v-for先循环再判断条件，如果只渲染一小部分也会导致重渲染的时候遍历整个列表，浪费性能

    • 在vue3中，v-if的优先级高于v-for

      由于先判断在循环，在判断的时候会因为变量还不存在而报错

    如果真的需要，最好的办法是定义function将列表过滤后再用v-for进行循环

    v-on的作用

    给标签绑定函数，可以缩写为@

    v-text和v-html解析文本

    v-text：用于解析文本，但不能解析标签，并且会覆盖元素内部原有的内容

    v-html：可以把带有标签的字符串，渲染成真正的 HTML 内容
